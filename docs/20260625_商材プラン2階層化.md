# 商材／プランの2階層化 実装メモ

作成日：2026-06-25

## 目的
今は「Hinova Biz スタンダード」「同 プレミアム」を別々の商材として登録しており、
LP URL・事業・把握方法などの共通情報が重複している。これを：

```
商材（サービス）  Hinova Biz / Hinova CRM   … LP URL・事業・把握方法（共通）
   └─ プラン       スタンダード / プレミアム  … 月額・報酬率・端数処理（プランごと）
```

の2階層にする。

## 方式（低リスク）
- **報酬率・月額はプラン側に残す** → 報酬計算ロジック（commission.js）は基本そのまま
- 新テーブル `services`（商材）に共通情報を持たせる
- 既存 `products` を「プラン」として扱い、`service_id` で商材に紐付ける
- `deals/leads/commissions/partner_commission_rules/partner_products` は **product_id（=プラン）参照のまま**（churn最小）

## DB変更（20260625100000_services_plans.sql）
- `services`（id, name, business, lp_url, detection_method, description, is_active, sort_order）
- `products.service_id` 追加（FK→services）
- データ移行：既存products を `商材名 - プラン名` で分割し、商材名ごとに service を作成、
  products.name をプラン名（スタンダード等）に更新、service_id を設定
- services の RLS（管理者=全、ログイン=select）

## コード変更
- Products.jsx → 商材（サービス）管理＋配下プラン管理に作り替え
- commission.js → お礼の product_name スナップショットを「商材名 プラン名」に
- Deals/Leads → 商材ごとにグルーピング表示（参照は従来通りプラン）
- PortalRefer → LP共有は services（商材）単位
- PortalReferrals / Customers → 「商材名 プラン名」で表示
- Partners → 担当商材・個別報酬率を商材でグルーピング表示
- 資料の事業フィルタ：プランに business を同期して従来動作を維持

## 移行対象データ（少量）
- products 4件（Biz/CRM × スタンダード/プレミアム）→ services 2 + plans 4
- leads（きーちゃん 4ライン）/ commissions は product_id 参照のまま有効
