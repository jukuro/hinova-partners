# Hinova Partners 報酬率ベース設計提案

作成日：2026年6月23日

---

## 1. 現状と変更の概要

### 現状の仕組み
- `products` テーブルの `commission_type`（fixed / rate / recurring）と `commission_value` で報酬を管理
- `partner_commission_rules` テーブルでパートナー個別の上書きが可能
- 「固定金額」「率」「継続報酬」が混在しており、管理が複雑

### 変更後の仕組み
- **全商材を「月額 × 報酬率」の一本化**
- 報酬率はパートナーランクによって自動加算
- 過去の報酬計算結果をイミュータブルな履歴として保存

---

## 2. 報酬計算ロジック

```
適用報酬率 = 以下の優先順位で決定：

① partner_product_rates に該当レコードあり → その custom_rate を使用
② なければ：商材の base_rate + パートナーランクの rate_addition
③ ランク未設定/審査中などで加算が無い → 商材の base_rate のみ（②で加算0と同義）

お礼金額 = floor(決済金額 × 適用報酬率 / 10) × 10  ← 10円単位切り捨て（デフォルト）
```

### 計算例

| 要素 | 値 |
|---|---|
| 商材 | Hinova CRM プレミアム |
| 月額 | ¥2,980 |
| 商材 base_rate | 17% |
| パートナーランク | Hinova Local Partner (+3%) |
| 適用報酬率 | 20% |
| 計算値 | ¥2,980 × 20% = ¥596 |
| 端数処理 | 10円切り捨て → **¥590** |

### 契約日による報酬率ロック（重要）

報酬率は **契約日（顧客の利用開始日）時点の率で確定し、その契約が続く限り固定** する。

- 契約成立時に、解決した報酬率を**その契約（deal）にスナップショット保存**する。
- 以降の毎月の報酬計算は、契約に保存された率を使う（毎月再解決しない）。
- パートナーのランク変更・商材の基本報酬率変更は、**変更日以降に成立した新規契約のみ**に適用され、既存契約の率は変わらない。
- 当然、確定済みの `reward_history` も不変（再計算しない）。

> これにより「ランク変更時に未払い履歴を再計算するか」という問題自体が発生しない。
> 過去・既存契約は契約日の率、新規契約だけ新しい率、という一貫した切り分けになる。

## 3. データベース設計

### 3-1. 変更・追加テーブル一覧

| テーブル | 変更内容 |
|---|---|
| `products` | 報酬関連カラムを新設計へ移行（後方互換のため旧カラムは残す） |
| `partner_ranks` | **新規作成**：ランク定義とランク加算率 |
| `partners` | `rank` カラムを `rank_id` (FK) に変更 |
| `partner_product_rates` | **新規作成**：パートナー×商材の個別報酬率 |
| `reward_history` | **新規作成**：報酬計算の不変履歴 |

### 3-2. `partner_ranks`（新規）

```sql
CREATE TABLE partner_ranks (
  id           uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name         text NOT NULL,           -- 表示名
  slug         text NOT NULL UNIQUE,    -- 内部識別子
  rate_addition numeric(5,2) NOT NULL DEFAULT 0, -- ランク加算率（%）
  sort_order   int  NOT NULL DEFAULT 0,
  created_at   timestamptz DEFAULT now()
);
```

**初期データ（3ランク）：**

仕様書（第10節・第17節）の用語方針に合わせ、ランク名は仕様書既存名を踏襲。
「代理店」表現は使わない。

| slug | name | rate_addition | 備考 |
|---|---|---|---|
| `partner` | Hinova Partner | +0% | デフォルト（審査通過時の初期ランク） |
| `local_partner` | Hinova Local Partner | +3% | 地域・業種内で継続実績あり |
| `trusted_partner` | Hinova Trusted Partner | +8% | 信頼性・継続性が高い |

> **Hinova Supporter / Hinova Guide は廃止**
> Supporter は「審査確認中」、Guide は「審査通過直後」を指しており、
> ランク（報酬段階）ではなく**ステータス**の性質が強いため、ランクから外す。
> 「審査中」は `partners.status` で管理する（下記 3-4）。
>
> **個別設定（partner_product_rates）がないパートナーの扱い**
> どのランクでも共通：`商材の基本報酬率 + ランク加算率`。
> 旧「特別パートナー」のような専用扱いは設けない（個別設定が必要な場合は
> partner_product_rates で商材ごとに上書きする）。
>
> **既存ランク（supporter / guide / partner / local_partner / trusted_partner）との対応**
> 移行時に以下のマッピングで一括UPDATE：
> - supporter → partner（※併せて status を `reviewing` に）
> - guide → partner
> - partner → partner
> - local_partner → local_partner
> - trusted_partner → trusted_partner
>
> 移行後、古い `rank`(text) カラムは削除。

### 3-3. `products` テーブル（変更）

追加するカラム：

```sql
ALTER TABLE products
  ADD COLUMN base_reward_rate  numeric(5,2),  -- 基本報酬率（%）例: 17.00
  ADD COLUMN max_reward_rate   numeric(5,2),  -- 最大報酬率（上限）
  ADD COLUMN rounding_rule     text NOT NULL DEFAULT 'floor_10';
  -- rounding_rule: 'floor_10'（10円切り捨て）| 'floor_100'（100円切り捨て）| 'round'（四捨五入）
```

> 旧カラム（`commission_type`, `commission_value`）はデータ保全のため当面残す。
> 新規商材は `base_reward_rate` を使用。既存商材は手動で移行。

---

### 3-4. `partners` テーブル（変更）

```sql
ALTER TABLE partners
  ADD COLUMN rank_id uuid REFERENCES partner_ranks(id),
  ADD COLUMN status  text NOT NULL DEFAULT 'reviewing';
  -- status: 'reviewing'（審査中）| 'active'（稼働中）| 'inactive'（停止）
-- 既存の rank(text) カラムは移行後に削除
```

> 「審査中」のパートナーは紹介・報酬の対象外。`status='active'` になって初めて
> ランクに応じた報酬率が適用される。

### 3-5. `partner_product_rates`（新規）

パートナー×商材の個別報酬率（最優先）：

```sql
CREATE TABLE partner_product_rates (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  partner_id  uuid NOT NULL REFERENCES partners(id) ON DELETE CASCADE,
  product_id  uuid NOT NULL REFERENCES products(id) ON DELETE CASCADE,
  custom_rate numeric(5,2) NOT NULL,  -- 個別報酬率（%）
  note        text,
  created_at  timestamptz DEFAULT now(),
  UNIQUE(partner_id, product_id)
);
```

---

### 3-6. `reward_history`（新規）

報酬計算の不変履歴（一度保存したら変更しない）：

```sql
CREATE TABLE reward_history (
  id               uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  period_month     text NOT NULL,        -- 対象月 例: '2026-06'
  partner_id       uuid NOT NULL REFERENCES partners(id),
  customer_name    text,                 -- 顧客名（スナップショット）
  product_id       uuid REFERENCES products(id),
  product_name     text NOT NULL,        -- 商材名（スナップショット）
  payment_amount   numeric(12,0) NOT NULL, -- 決済金額
  applied_rate     numeric(5,2) NOT NULL,  -- 適用された報酬率（%）
  reward_amount    numeric(12,0) NOT NULL, -- 計算されたお礼金額
  calculation_basis jsonb NOT NULL,        -- 計算根拠（後述）
  payment_status   text NOT NULL DEFAULT 'pending',
  -- 'pending'（未払い）| 'paid'（支払済み）| 'cancelled'（取消）※承認ステップは設けない
  paid_at          timestamptz,
  note             text,
  created_at       timestamptz DEFAULT now()
);
```

> 支払い承認ステップは不要との決定により、ステータスは
> **未払い → 支払済み** の2段階のみ（取消は例外運用）。

**`calculation_basis` の保存内容（例）：**

```json
{
  "base_rate": 17,
  "rank_name": "Hinova Local Partner",
  "rank_addition": 3,
  "custom_rate": null,
  "applied_rate": 20,
  "rounding_rule": "floor_10",
  "raw_amount": 596,
  "final_amount": 590,
  "resolved_by": "rank",
  "locked_at": "2026-06-23"
}
```

- `resolved_by`：`"custom"`（①個別設定）/ `"rank"`（②ランク加算）/ `"base"`（③基本率のみ）
- `locked_at`：率を確定した契約日（契約日ロックの根拠）

---

### 3-7. `deals`（契約）への報酬率スナップショット（変更）

契約日ロックを実現するため、契約成立時に解決した報酬率を deal に保存する。

```sql
ALTER TABLE deals
  ADD COLUMN locked_reward_rate numeric(5,2),  -- 契約日に確定した適用報酬率（%）
  ADD COLUMN locked_rate_basis  jsonb,         -- 確定時の計算根拠スナップショット
  ADD COLUMN contract_date      date;          -- 契約日（利用開始日）
```

毎月の報酬計算は `locked_reward_rate` を使用し、ランク・基本率を再解決しない。
（deals テーブルが未整備の場合は、契約相当のレコードに同等カラムを追加する）

## 4. ビジネスロジック（`commission.js` 変更）

```js
// 報酬率を解決する（①→②→③の優先順位）
export async function resolveRewardRate(partnerId, productId) {
  // ① パートナー個別 × 商材個別
  const { data: custom } = await supabase
    .from('partner_product_rates')
    .select('custom_rate')
    .eq('partner_id', partnerId)
    .eq('product_id', productId)
    .maybeSingle();
  if (custom) return { rate: custom.custom_rate, resolved_by: 'custom', rank_addition: 0 };

  // ② ランク加算率 + 商材基本率（③ 加算0は基本率のみと同義）
  const [{ data: partner }, { data: product }] = await Promise.all([
    supabase.from('partners').select('rank_id, status, partner_ranks(rate_addition, name)').eq('id', partnerId).maybeSingle(),
    supabase.from('products').select('base_reward_rate, max_reward_rate').eq('id', productId).maybeSingle(),
  ]);

  const base = product?.base_reward_rate ?? 0;
  const rank = partner?.partner_ranks;
  // 審査中は加算なし（基本率のみ）
  const addition = partner?.status === 'active' ? (rank?.rate_addition ?? 0) : 0;

  const rate = base + addition;
  const max = product?.max_reward_rate;
  const applied = max ? Math.min(rate, max) : rate;

  return {
    rate: applied,
    resolved_by: addition > 0 ? 'rank' : 'base',
    base_rate: base,
    rank_name: rank?.name ?? null,
    rank_addition: addition,
  };
}

// 端数処理
export function applyRounding(amount, rule = 'floor_10') {
  if (rule === 'floor_10')  return Math.floor(amount / 10) * 10;
  if (rule === 'floor_100') return Math.floor(amount / 100) * 100;
  if (rule === 'round')     return Math.round(amount);
  return amount;
}

// 報酬金額を計算
export function calcReward(paymentAmount, rate, roundingRule = 'floor_10') {
  const raw = paymentAmount * (rate / 100);
  return { raw, final: applyRounding(raw, roundingRule) };
}

// 契約成立時に報酬率を確定（契約日ロック）
// → 解決した率を deal にスナップショット保存して以降固定する
export async function lockRewardRateForDeal(dealId, partnerId, productId, contractDate) {
  const resolved = await resolveRewardRate(partnerId, productId);
  await supabase.from('deals').update({
    locked_reward_rate: resolved.rate,
    locked_rate_basis:  { ...resolved, locked_at: contractDate },
    contract_date:      contractDate,
  }).eq('id', dealId);
  return resolved;
}

// 毎月の報酬を確定して履歴に保存（率は deal にロック済みのものを使う）
export async function recordMonthlyReward({ deal, product, periodMonth, paymentAmount }) {
  const rate = deal.locked_reward_rate;              // 契約日ロックの率
  const rule = product.rounding_rule || 'floor_10';
  const { raw, final } = calcReward(paymentAmount, rate, rule);
  await supabase.from('reward_history').insert({
    period_month:   periodMonth,
    partner_id:     deal.partner_id,
    customer_name:  deal.customer_name,
    product_id:     product.id,
    product_name:   product.name,
    payment_amount: paymentAmount,
    applied_rate:   rate,
    reward_amount:  final,
    calculation_basis: { ...deal.locked_rate_basis, rounding_rule: rule, raw_amount: raw, final_amount: final },
    payment_status: 'pending',
  });
}
```
